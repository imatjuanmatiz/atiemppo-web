## Cuerpo base
En muchas empresas de transporte, el valor de referencia de una ruta sigue viviendo en consultas manuales, hojas sueltas o personas que cargan el conocimiento en la cabeza.

Ese modelo ya no da la talla.

Por eso en ATIEMPPO estamos convirtiendo nuestra **API SICETAC** en un producto: una capa moderna para que las empresas conecten sus sistemas, sus agentes y sus equipos al cálculo de costos de referencia de manera más rápida, más consistente y mucho más útil para la operación diaria.

La idea no es solo exponer un endpoint. La idea es habilitar una capacidad nueva dentro de la empresa.

## Qué permite esta capa

- conectar el TMS, CRM, ERP o herramientas internas a consultas de valor SICETAC
- alimentar agentes conversacionales para responder preguntas de rutas en lenguaje natural
- llevar costos de referencia a equipos comerciales que necesitan reaccionar más rápido
- dar más contexto a operación para seguimiento de viajes, rentabilidad y control
- ayudar al área financiera a tener una base más clara para seguimiento de costos, presupuestos y conversaciones de margen

## Por qué importa

Cuando el valor de referencia no fluye bien dentro de la empresa, cada área termina trabajando con versiones distintas del mismo problema.

Comercial cotiza con una lógica.
Operación revisa con otra.
Finanzas analiza después, cuando ya es tarde.

Una capa de integración bien hecha ayuda a reducir esa fricción.

No reemplaza el criterio de negocio. Pero sí pone una base común para que las decisiones se tomen con más velocidad y mejor trazabilidad.

## Lo que hace distinto este enfoque

No estamos pensando la API como un producto aislado para desarrolladores.

La estamos pensando como una infraestructura útil para empresas de transporte que quieren:

- conectar sistemas
- habilitar agentes
- automatizar consultas recurrentes
- modernizar la forma en que sus equipos entienden el costo de una ruta

Además, esta capa ya nace con dos ventajas prácticas:

- puede responder por API para integraciones directas
- puede servir como backend para agentes y experiencias conversacionales

Es decir: sirve tanto para sistemas como para interfaces nuevas de trabajo.

## Para quién hace sentido

Esta capa hace sentido si en tu empresa hoy existe al menos una de estas situaciones:

- el equipo comercial pierde tiempo consultando rutas de forma manual
- operación necesita contrastar rápido referencias de costo
- finanzas quiere más consistencia en el seguimiento de costos y proyectos
- estás explorando agentes internos o asistentes de IA conectados a datos reales
- quieres que el valor SICETAC deje de ser una consulta aislada y pase a ser una capacidad integrada
